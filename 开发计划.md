### **项目名称**
CryptoPilot（或自定义）—— 币安实盘自动交易系统

### **项目概述**
开发一个面向币安现货与合约（根据需求可先支持合约）的自动化交易程序，**直接运行在实盘环境**。程序需具备完整的策略执行、仓位管理、风险控制、行情监控、通知推送、日志记录及数据持久化功能。不使用 Docker，支持本地直接部署运行（如 Python 虚拟环境或裸机部署）。要求高可用、低延迟，能够稳定处理 WebSocket 实时数据流，严格保护 API 密钥安全。

---

## **1. 技术栈建议**
- **语言**：Python 3.10+（生态丰富，开发效率高）
- **WebSocket 客户端**：`websockets` 或 `unicorn-binance-websocket-api`（专为币安优化）
- **REST API 客户端**：`python-binance`（或直接使用 `requests` + 自封装签名逻辑）
- **数据库**：SQLite（轻量，无需额外服务）
- **日志**：`loguru`（简洁高效，支持自动轮转）
- **通知**：Telegram Bot API（`python-telegram-bot`）
- **配置管理**：`.env` + `pydantic-settings`，API 密钥加密存储（如 `cryptography` 库 AES 加密）
- **任务调度**：`apscheduler` 或内置 `asyncio` 事件循环
- **Web 管理界面**：`FastAPI` + 简单前端，仅查看状态和手动干预

## **2. 核心功能模块与需求**

### **2.1 行情监控与扫描模块**
- 通过 **WebSocket** 订阅币安实时行情（`kline`、`ticker`、`depth` 等），支持多交易对并发监听。
- 支持自定义扫描逻辑：如价格突破、成交量异常、均线交叉、RSI 超买超卖等信号检测。
- 行情数据实时推送到策略引擎，支持毫秒级响应（优化事件总线和内存数据结构）。
- **重连机制**：WebSocket 断线自动重连，重连后自动重新订阅，确保不丢数据。

### **2.2 策略引擎**
- 策略与核心交易逻辑分离，采用 **策略模板/接口** 设计，便于扩展新策略。
- 内置示例策略（如双均线趋势、网格震荡、马丁逆势加仓等），允许参数化配置。
- 策略生命周期：`on_init` -> `on_tick`/`on_kline` -> `on_signal`，支持挂单、市价单、限价单。
- 策略可同时运行多实例，每个实例绑定一个交易对和一套参数，互不干扰。
- 策略内部集成 **仓位计算**：根据账户余额、杠杆、风险百分比动态计算下单量。

### **2.3 开平仓与交易执行模块**
- 封装币安 API 的下单、撤单、查询订单/持仓接口（现货和合约分开）。
- 支持 **市价单**、**限价单**、**止盈止损单**、**OCO 订单** 等。
- **订单状态跟踪**：维护本地订单簿，处理部分成交、完全成交、取消等状态，确保与交易所同步。
- **开仓前检查**：余额充足、杠杆正确、持仓模式（单向/双向）、最小交易量限制、价格精度等。
- **平仓逻辑**：全平、部分平仓、反手开仓，支持手动紧急平仓（通过 Telegram 指令或 Web 面板）。

### **2.4 止盈止损与保护机制**
- **硬止损**：开单时自动提交币安原生的止损单（`stopMarket`/`stopLimit`），即使程序崩溃也生效。
- **移动止损/保护**：根据行情变动逐步上移止损价，锁定更多利润（程序内存跟踪，定期调整）。
- **锁利逻辑**：当浮盈达到指定条件，自动平掉部分仓位保证利润，或设置保本止损。
- **风控熔断**：当日亏损超过阈值（如净值 2%），立即停止所有策略并平掉所有仓位，发送严重告警。

### **2.5 账户与仓位保护**
- 实时监控账户保证金率、强平价格，接近强平线时自动减仓或追加保证金提醒。
- 最大同时持仓数限制，单币种最大敞口控制。
- 杠杆动态调整：策略可指定初始杠杆，避免过高风险。
- **API 权限控制**：建议仅开启交易权限，关闭提币权限，IP 白名单绑定。

### **2.6 通知推送模块**
- 支持 **Telegram
- 推送事件：策略开仓/平仓信号、成交回报、止盈止损触发、异常错误（如连续掉线、余额不足、风控熔断）、每日净值总结。
- 交互式通知：Telegram Bot 可接收指令，如 `/status`、`/close_all`、`/pause`、`/resume` 等，需身份验证。

### **2.7 日志记录**
- 分级日志：DEBUG（开发调试）、INFO（关键节点）、WARNING（可恢复异常）、ERROR（需人工介入）。
- 日志文件每日轮转，保留至少 30 天，包含时间戳、模块名、日志级别、详细信息。
- 关键交易日志单独记录到 `trade.log`，方便审计和复盘。

### **2.8 订单与交易数据持久化**
- 数据库保存：所有已提交订单（含未成交、已成交、已取消）、成交明细、策略持仓变更历史、账户快照（定时）。
- 表结构设计示例：
  - `orders`：`id, symbol, strategy_name, side, type, price, qty, status, timestamp, client_order_id, exchange_order_id, pos_side`
  - `fills`：`id, order_id, price, qty, commission, timestamp`
  - `positions`：`id, symbol, side, qty, entry_price, leverage, liquidation_price, timestamp`
  - `accounts`：`timestamp, total_balance, available_balance, unrealized_pnl`
- 支持事后查询，生成简单交易报表（盈亏、胜率、最大回撤等），便于分析策略表现。

### **2.9 系统运行与监控**
- 主进程需提供 **健康检查 HTTP 接口**（如 `http://localhost:1688/health`），返回运行状态、各策略状态、队列积压等。
- 进程守护：可用 `supervisor` 或 `systemd` 自动重启，崩溃后 5 秒内拉起。
- 无需 Docker，提供一行启动脚本 `start.sh`（含环境检查、依赖安装、启动命令）。

## **3. 非功能需求**
- **性能**：WS 延迟 < 100ms，订单响应 < 500ms，多策略并行不阻塞。
- **安全**：API Key/Secret 必须加密存储，运行内存中解密使用，日志绝不记录 Secret。
- **可用性**：7x24 稳定运行，处理网络闪断、交易所临时维护等异常。
- **可配置性**：所有参数（策略、风控、通知等）通过配置文件和环境变量管理，无需改动代码。
- **可扩展**：新增策略仅需继承基类并实现信号方法，方便后期扩展。

## **4. 架构概览**
```
[配置管理] 
    ↓
[WebSocket行情] → [行情缓存/事件总线] → [策略引擎] → [信号队列] → [订单执行器] → [币安API]
                     ↕                               ↕
              [数据库(持久化)]                [通知模块]
```
采用异步事件驱动架构，所有模块间通过消息队列（可选 `asyncio.Queue`）解耦，便于独立扩展和维护。

## **5. 开发流程建议**
1. **搭建基础框架**：配置加载、日志、数据库、加密存储 API 密钥。
2. **实现行情模块**：WS 连接、重连、数据解析，写入共享缓存。
3. **开发订单执行器**：封装下单、撤单、查询接口，模拟单测试（但实盘配置，开始时极小量）。
4. **策略基类与示例策略**：实现简单的双均线策略，能跑通开平仓闭环。
5. **风控与保护机制**：逐步加入止损、移动止损、熔断，完善仓位计算。
6. **通知与交互**：接 Telegram Bot，支持指令。
7. **持久化完善**：行情数据存储、报表生成（可后期迭代）。
8. **容错与守护**：进程守护、健康检查、异常恢复。

---

## **6. 特别注意与风险提示**
- **首次实盘必须以极小金额测试**（建议 10 USDT 以内），验证所有流程无误后再放大。
- 务必区分现货和合约的订单参数（如 `positionSide`、`reduceOnly` 等字段），避免误开反向仓位。
- 币安 API 有频率限制（如 WebSocket 连接数、每分钟下单量），程序需实现限速器（`RateLimiter`）。
- 所有资金操作前必须二次核对数量、价格精度，四舍五入为交易所允许的 step size。
- 当策略出现连续报错，应主动暂停该策略并通知，而非无限重试。
